에라토스테네스의 체

작성일 2020-10-08 수정일 2020-10-10 카테고리 algorithm Disqus:

개요

nnn 이하의 모든 소수를 찾으려고 합니다.
1부터 nnn까지 순회하면서 각 수에 대해 간단히 소수판별을 하는 방법을 생각해 볼 수 있습니다.

isPrime(x) 함수는 2부터 x\sqrt{x}x​까지만 나누어 보는 시행을 하여 소수판별이 가능합니다.
따라서, 이 코드의 경우 전체 시간복잡도는 O(nn)O(n \sqrt{n})O(nn​)가 됩니다.
이걸 개선해 보도록 합시다.

에라토스트네스의 체

알고리즘 자체는 매우 간단합니다.

  1. 1부터 nnn까지에 해당하는 공간을 잡는다.
  2. 1은 소수가 아니므로 지운다.
  3. 2는 지워지지 않았으므로 체크하고, 2를 제외한 2의 배수를 모두 지운다.
  4. 3은 지워지지 않았으므로 체크하고, 3을 제외한 3의 배수를 모두 지운다.
  5. 4는 지워졌으므로 넘어간다.
  6. nnn까지 이 과정을 반복한다.

연산량을 계산하면 다음과 같습니다.

∑p≤nnp=n⋅∑p≤n1p\sum_{p \leq n} \frac{n}{p} = n\cdot \sum_{p \leq n} \frac{1}{p} p≤n∑​pn​=n⋅p≤n∑​p1​

이때,

∑p≤n1p≈12+∑k=2nln⁡n1kln⁡k\sum_{p \leq n} \frac{1}{p} \thickapprox \frac{1}{2} + \sum_{k=2}^{\frac{n}{\ln n}} \frac{1}{k \ln k} p≤n∑​p1​≈21​+k=2∑lnnn​​klnk1​

임이 알려져 있으므로, 식을 계산해보면

∑k=2nln⁡n1kln⁡k≈∫2nln⁡n1kln⁡kdk=ln⁡ln⁡nln⁡n−ln⁡ln⁡2=ln⁡(ln⁡n−ln⁡ln⁡n)−ln⁡ln⁡2≈ln⁡ln⁡n\begin{aligned} \sum_{k=2}^{\frac{n}{\ln n}} \frac{1}{k \ln k} &\thickapprox \int_2^{\frac{n}{\ln n}} \frac{1}{k \ln k} dk\\ &= \ln\ln\frac{n}{\ln n} - \ln\ln 2\\ &= \ln(\ln n - \ln\ln n) - \ln\ln 2\\ &\thickapprox \ln\ln n \end{aligned} k=2∑lnnn​​klnk1​​≈∫2lnnn​​klnk1​dk=lnlnlnnn​−lnln2=ln(lnn−lnlnn)−lnln2≈lnlnn​

이 됩니다.
따라서, 전체 시간복잡도는 O(nlog⁡log⁡n)O(n \log \log n)O(nloglogn)이 됩니다.

구현

n이 소수인지 확인하려면 sieve[n]가 false인지 보면 됩니다.

그 외 정보

여러가지 최적화 기법

sqrt까지만 체로 거르기

첫번째 for문에서 i2^22이 nnn이상이라면 두번째 for문에서 sieve의 갱신은 이루어지지 않으므로 첫번째 for문의 범위를 n\sqrt{n}n​로 줄이는 기법이 있습니다.

O(nlog⁡log⁡n)O(n \log \log \sqrt{n})O(nloglogn​)의 시간복잡도가 되지만, 결국 O(nlog⁡log⁡n)O(n \log \log n)O(nloglogn)과 동일합니다.

홀수만 따지기

2를 제외한 짝수는 모두 합성수이므로, 짝수를 처음부터 고려하지 않는 기법이 있습니다.
이 방법을 사용하면 메모리와 시간 모두 절반으로 줄일 수 있습니다.

홀수 n이 소수인지 확인하려면 sieve[n >> 1]가 false인지 보면 됩니다.

비트단위 압축

bool은 보통 1바이트입니다(C++ 표준에서는 1바이트임을 보장하지 않습니다!).
true/false는 1비트로도 표현이 가능하기 때문에, 비트로 표현하면 메모리를 18\frac{1}{8}81​배 절약할 수 있습니다.

n이 소수인지 확인하려면 sieve[n >> 3] & 1 << (n & 7)이 0인지 보면 됩니다.

구간 체

공간복잡도를 O(n+S)O(\sqrt{n} + S)O(n​+S)으로 줄이는 기법입니다(SSS는 상수).
[1, nnn] 범위의 수를 소수인지 판별하기 위해서는 [1, n\sqrt{n}n​] 범위의 소수만 가지고 있으면 충분하기 때문에, 이를 이용해서 체로 걸러낼 수 있습니다.

SSS가 너무 작지만 않다면 위 코드의 시간복잡도는 O(nlog⁡log⁡n)O(n \log \log n)O(nloglogn)으로 동일합니다.
따라서 상수 SSS를 몇으로 잡는지가 꽤 중요한데, 많은 실험에 의해 [10410^4104, 10510^5105] 사이의 수가 가장 적절하다고 합니다.

[L, R] 범위의 소수 찾기

R이 101210^{12}1012처럼 매우 크고, R-L+1이 비교적 작다고 해봅시다.
정말 매우 작은 경우에는 직접 L부터 R까지 순회하면서 소수판별하는 방법이 먹힙니다.
하지만 R-L+1이 10710^7107 정도라면 그 방법은 좀 부담스럽죠.

이 문제를 풀 때는 위에서 소개한 구간 체 아이디어를 사용합니다.
R\sqrt{R}R​ 이하의 모든 소수를 만들어서 [L, R] 구간의 소수를 체크하면 됩니다.

L+kL+kL+k가 소수인지 확인하려면 sieve[k]가 0인지 보면 됩니다.
위 코드의 시간복잡도는 O((R−L+1)log⁡log⁡R+Rlog⁡log⁡R)O((R - L + 1) \log \log R + \sqrt{R} \log \log \sqrt{R})O((R−L+1)loglogR+R​loglogR​)입니다.

조금 느리지만 더 간단한 구현도 있습니다.

소수 판별은 위와 같습니다.
시간복잡도는 O((R−L+1)log⁡R+R)O((R - L + 1) \log R + \sqrt{R})O((R−L+1)logR+R​)이지만, 여전히 쓸 만한 시간복잡도입니다.

선형 체

선형 체는 이름에서 알 수 있듯이 nnn 이하의 소수를 구하는 데 O(n)O(n)O(n)밖에 걸리지 않습니다.
하지만 에라토스테네스의 체처럼 메모리 관련해서 최적화 기법을 적용할 수 없습니다.
따라서, 보통의 문제에선 nnn이 10710^7107 이하일 때나 써먹기에 적절하겠죠.
